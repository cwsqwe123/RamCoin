# AgentPay SDK version 0.3.1 release notes

**Release date:** 2026-04-13  
**Current version:** v0.3.1

These notes cover what is new in AgentPay SDK `v0.3.1` relative to `v0.3.0`.

For EIP-712 signing, MPP session improvements, and shared-config policy refresh, see [v0.3.0 release notes](./v0.3.0.md).

## Release summary

AgentPay SDK `v0.3.1` brings full managed-wallet support to Linux, extending the admin lifecycle and daemon management that was previously macOS-only to Linux systems using `systemd` and the Linux Secret Service API. It also fixes macOS installer compatibility with the default system Bash (3.2) and deduplicates CI workflow runs.

## What's new in v0.3.1

### Managed Linux wallet support

- added managed Linux wallet lifecycle powered by `systemd` services for daemon management and the Linux Secret Service D-Bus API (via `libsecret`) for credential storage
- enabled `agentpay admin setup`, `agentpay admin tui`, `agentpay admin reset`, and `agentpay admin uninstall` on Linux alongside macOS
- added `systemd` service unit templates and helper scripts:
  - `install-system-daemon.sh` — installs and enables the daemon as a system service
  - `uninstall-system-daemon.sh` — cleanly removes the daemon service and state
  - `run-agentpay-daemon.sh` — service entry point for `systemd`
  - `agentpay-daemon-password-helper.sh` — retrieves encrypted-state passwords from the Secret Service at daemon start

### Installer and platform updates

- updated the packaged installer and runtime guidance to describe the Linux managed-wallet path, replacing the previous runtime-only install flow
- source installs now support both macOS and Linux for the CLI and runtime binaries
- added packaged Linux installer bundles alongside the existing macOS bundles
- added Linux installer smoke and build coverage to the release workflow

### Credential storage and status

- added Linux Secret Service-backed agent auth storage as the platform-native credential backend
- extended shared-config/live-apply and wallet status flows to report Linux credential storage and managed daemon state correctly

### macOS installer compatibility

- added `configure_bash_compatibility()` shim so the one-click installer works under macOS default `/bin/bash` (GNU Bash 3.2)
- replaced dynamic file descriptor allocation (`exec {PROMPT_IN_FD}<>/dev/tty`) with a fixed fd `3` fallback for Bash < 4
- made arrow-key escape-sequence read timeouts configurable and Bash 3.2-safe

### CI improvements

- changed CI concurrency key to deduplicate matching `push` and `pull_request` workflow runs on the same branch

### Platform boundary hardening

- macOS-only managed-wallet commands now fail fast on Linux when the Linux managed path is not available
- macOS-only helper binaries are no longer installed by Linux source installs
- daemon-socket recovery guidance now points Linux users to explicit source-managed sockets instead of `agentpay admin setup`
